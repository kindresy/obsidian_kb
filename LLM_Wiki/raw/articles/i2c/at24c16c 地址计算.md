AT24C16C 是 16Kbit I2C EEPROM，也就是：

```text
16 Kbit = 2048 Byte = 2 KB
地址范围：0x000 ~ 0x7ff
组织方式：2048 x 8-bit
页大小：16 Byte
```

它和 AT24C32/64/256 这类大容量 EEPROM 一个关键区别是：AT24C16C 不发送 2-byte word address，而是把 11-bit 存储地址拆开。

**1. 地址拆分**

AT24C16C 需要 11-bit memory address：

```text
A10 A9 A8 A7 A6 A5 A4 A3 A2 A1 A0
```

但 I2C 访问时只发送 1 个 word address byte，也就是低 8 位：

```text
A7..A0
```

高 3 位：

```text
A10..A8
```

被放进 I2C device address 里。

datasheet 的 device address byte 格式是：

```text
bit7 bit6 bit5 bit4 bit3 bit2 bit1 bit0
 1    0    1    0   A10  A9   A8  R/W
```

所以 7-bit slave address 是：

```text
1010 A10 A9 A8
```

换成代码就是：

```c
dev = 0x50 | ((offset & 0x700) >> 8);
word_addr = offset & 0xff;
```

这正是当前代码里的宏：

```c
#define I2C_TEST_AT24C16C_SLAVE_ADDR  0x50U
#define I2C_TEST_AT24C16C_DEV_ADDR(a) \
	(I2C_TEST_AT24C16C_SLAVE_ADDR | (((a) & 0x700U) >> 8))
#define I2C_TEST_WORD_ADDR_L(a)       ((a) & 0xFFU)
```

**2. 地址例子**

| EEPROM offset | A10:A8 | 7-bit dev addr | word addr |
|---:|---:|---:|---:|
| `0x000` | `0` | `0x50` | `0x00` |
| `0x0ff` | `0` | `0x50` | `0xff` |
| `0x100` | `1` | `0x51` | `0x00` |
| `0x160` | `1` | `0x51` | `0x60` |
| `0x200` | `2` | `0x52` | `0x00` |
| `0x700` | `7` | `0x57` | `0x00` |

所以 AT24C16C 在总线上看起来会响应 `0x50..0x57` 这 8 个 7-bit 地址，每个地址对应 256B block。

**3. Byte Write**

写 1 个字节：

```text
START
Device Address + W
ACK
Word Address A7..A0
ACK
Data Byte
ACK
STOP
内部写周期
```

例如写 offset `0x160`：

```text
dev addr = 0x51
word addr = 0x60
```

流程：

```text
START
0x51 + Write
0x60
data
STOP
```

**4. Page Write**

AT24C16C 页大小是 16B。页写流程：

```text
START
Device Address + W
ACK
Word Address
ACK
Data0
ACK
Data1
ACK
...
DataN
ACK
STOP
内部写周期
```

注意页边界：页内低 4 bit 地址会自动递增。如果一次写超过当前 16B page 边界，地址会在同一页内回卷，不会自动跨到下一页。

例如从 `0x16c` 写 16B，实际会覆盖：

```text
0x16c..0x16f，然后回卷到 0x160..0x16b
```

所以可靠写法是：每次 page write 不跨 16B 页边界。

**5. Current Address Read**

EEPROM 内部有当前地址指针。current address read 不先写 word address：

```text
START
Device Address + R
ACK
Data
NACK
STOP
```

读的是内部当前地址指针指向的位置，读完后地址指针递增。

**6. Random Read**

随机读某个 offset，需要先“伪写”地址，再 repeated start 读数据：

```text
START
Device Address + W
ACK
Word Address
ACK
START / repeated START
Device Address + R
ACK
Data
NACK
STOP
```

例如读 `0x160`：

```text
START
0x51 + W
0x60
RESTART
0x51 + R
read data
STOP
```

**7. Sequential Read**

随机读启动后，可以连续读多个字节：

```text
START
Device Address + W
Word Address
RESTART
Device Address + R
Data0 ACK
Data1 ACK
...
DataN NACK
STOP
```

读地址会自动递增。读操作通常可以跨页继续递增，直到器件地址空间末尾后回卷，具体看 datasheet 细节和访问长度控制。

**8. 为什么你的 demo offset 会访问 0x51**

当前代码：

```c
#define I2C_TEST_DEMO_OFFSET 0x160U
```

所以：

```text
offset = 0x160
A10:A8 = 1
dev addr = 0x50 | 1 = 0x51
word addr = 0x60
```

因此 demo 实际访问的是 slave address `0x51`。如果你的板子上 EEPROM 不是 AT24C16C，而是 AT24C02/04/08，或者地址引脚/器件映射不是这种方式，那么 `0x51` 可能不会 ACK。

参考：Microchip AT24C16C datasheet 说明该器件为 16-Kbit、2K x 8 组织，device address 中包含 word address 高位 A10/A9/A8，页大小为 16 bytes。来源：Microchip AT24C16C datasheet 和产品页。
